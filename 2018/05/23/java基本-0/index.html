  • Q：简述一下JVM加载class文件的原理机制
    A：
    加载：class加载到内存，创建字节数组读入.class文件，生成对象

    连接：验证，准备（static变量分配内存并设置初始值），解析（符号引用替换为直接引用）

    初始化：1）.父类没有初始化，初始化父类 2）.有初始化语句就执行初始化语句

    加载器：Bootstrap（加载JVM核心库）,Extension（系统属性指定目录加载类库，父类的Bootstrap），System（classpatch环境变量加载）

  • Q：抽象类（abstract class）和接口（interface）的区别

    A：

    共同：都不能够实例化，类（非抽象类）继承他们都要实现所有抽象方法

    不同：接口比抽象类更抽象，抽象类可以定义构造器，可有抽象方法和具体方法，接口不行

    抽象类中成员可以是private、default，protected，public的，接口都是public

    抽象类可以定义成员变量，接口中成员变量是常量，

    抽象方法的类必须被声明为抽象类，而抽象类未必要有抽象方法

    abstract和 static（native）矛盾

  • Q：如何实现对象的克隆

    A：

    1. 实现Clonable接口并重写Object类中的clone（）方法

    2. 实现Serializable接口，通过对象的序列化和反序列化实现克隆

    优于Object的clone方法，可以在编译的时候抛出异常

‘’’
import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;

public class MyUtil {

private MyUtil() {
    throw new AssertionError();
}

public static <T> T clone(T obj) throws Exception {
    ByteArrayOutputStream bout = new ByteArrayOutputStream();
    ObjectOutputStream oos = new ObjectOutputStream(bout);
    oos.writeObject(obj);

    ByteArrayInputStream bin = new ByteArrayInputStream(bout.toByteArray());
    ObjectInputStream ois = new ObjectInputStream(bin);
    return (T) ois.readObject();

    // 说明：调用ByteArrayInputStream或ByteArrayOutputStream对象的close方法没有任何意义
    // 这两个基于内存的流只要垃圾回收器清理对象就能够释放资源，这一点不同于对外部资源（如文件流）的释放
}

}

    final值不能变，类，方法不能继承
